### 1.2 Simulating Physics in Gazebo

**Learning Objectives:**
- Understand the architecture of Gazebo as a physics simulator.
- Learn how to simulate gravity, collisions, and sensor data.
- Explore basic robot models and world setups in Gazebo.

**Gazebo Overview:**  
Gazebo is an open-source 3D robotics simulator that provides realistic environments for testing and validating robot designs and behaviors. It is tightly integrated with **ROS 2**, which allows for real-time communication between simulation and control nodes.

**Key Features for Physics Simulation:**
- **Gravity Simulation:** Accurate modeling of gravitational forces on robot bodies and components.  
- **Collision Detection:** Supports detection of collisions between robot parts, environment objects, and obstacles.  
- **Rigid Body Dynamics:** Uses physics engines (ODE, Bullet, Simbody) to simulate movement, joint constraints, and inertia.  
- **Sensor Simulation:** Emulates sensors like LiDAR, depth cameras, IMUs, and contact sensors.  
- **Plugin System:** Extend functionality using custom Gazebo plugins for sensors, controllers, or environmental interactions.

**Gazebo Workflow for Physics Simulation:**
1. **Create a World:** Define the environment (ground plane, obstacles, lighting, textures).  
2. **Add Robots:** Import URDF/XACRO robot models and configure physical parameters (mass, inertia, joint limits).  
3. **Configure Physics Engine:** Select engine type (ODE, Bullet, Simbody) and set simulation parameters (time step, real-time factor).  
4. **Simulate Sensors:** Attach simulated sensors to robots to emulate real-world perception.  
5. **Run Simulation:** Launch the simulation using `gazebo` or via ROS 2 launch files.  
6. **Observe & Record:** Monitor robot behavior, debug collisions, collect sensor data for training or analysis.

### Tips for Effective Simulation:

Ensure realistic mass, friction, and damping values in your URDF models.

Use lightweight models for faster simulation when testing algorithms.

Record simulation logs for reproducibility and training AI models.

Start with simple environments and gradually increase complexity.
